(?) Is JAVA a Pass by Value or a Pass by Reference programming language?

Java is a pass-by-value programming language. 
When passing objects, the value of the reference is copied, so the method can modify the object's contents but cannot change the original reference.

(?) After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two
objects still remain?

Because Java passes object references by value. The swap method only swaps copies of the references, not the original objects.

(?) After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is
the title of the JungleDVD changed?

Because the method modifies the internal state of the object through its reference, which affects the original object.
